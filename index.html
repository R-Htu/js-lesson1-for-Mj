Lesson 1 · Basics

Variables

Variables are containers that store data. In modern JavaScript we use let and const.

📖 How it works

const — value never changes (most of the time, use this).

let — value can change later.

var — old way, avoid it.

const name = "John";       // cannot reassign
let age = 20;             // can reassign later
age = 21;               // ✅ works because it's let

const isStudent = true;   // boolean

console.log(name, age);  // John 21

Lesson 2 · Basics

Conditions

Conditions let your code make decisions — run different code depending on whether something is true or false.

📖 How it works

if runs the block when the condition is true.

else if checks another condition if the first was false.

else runs when none of the above matched.

const age = 20;

if (age >= 18) {
  console.log("Adult");
} else if (age >= 13) {
  console.log("Teenager");
} else {
  console.log("Child");
}
// Output: Adult

Lesson 3 · Basics

For Loop

A for loop repeats a block of code a set number of times — great for counting or going through a list.

📖 How it works

for (start; condition; step) — the loop runs as long as the condition is true.

Use for...of to loop through arrays more simply.

// Classic for loop
for (let i = 0; i < 3; i++) {
  console.log(i); // 0, 1, 2
}

// for...of (easier for arrays)
const fruits = ["🍎 Apple", "🍌 Banana", "🍇 Grape"];

for (const fruit of fruits) {
  console.log(fruit);
}

Lesson 4 · Basics

Functions

Functions are reusable blocks of code. Give them a name, call them whenever you need them.

📖 How it works

A function can take parameters (inputs) and return a result.

Arrow functions () => {} are a shorter modern syntax.

// Regular function
function greet(name) {
  return `Hello, ${name}!`;
}

// Arrow function (same thing, shorter)
const greet = (name) => `Hello, ${name}!`;

console.log(greet("John")); // Hello, John!

Lesson 5 · Arrays

Array Basics

An array is a list of items stored in one variable. You can add, remove, and access items by their position.

📖 How it works

Items are numbered starting at 0 — this is called the index.

Common methods: .push() add to end, .pop() remove from end, .length count items.

const fruits = ["Apple", "Banana", "Grape"];

fruits[0];         // "Apple"  (first item)
fruits[2];         // "Grape"  (third item)
fruits.length;     // 3

fruits.push("Mango");  // adds to end → 4 items
fruits.pop();         // removes last → "Mango" gone

Lesson 6 · Arrays

.map()

.map() goes through every item in an array and transforms it — returning a brand new array.

📖 How it works

The original array is never changed. .map() always returns a new array of the same length.

Think of it as: "do this to every item, give me the results."

const prices = [10, 20, 30];

// Double every price
const doubled = prices.map((price) => price * 2);
console.log(doubled); // [20, 40, 60]

// Add $ symbol to each
const names = ["john", "sarah"];
const upper = names.map((n) => n.toUpperCase());
// ["JOHN", "SARAH"]

Lesson 7 · Arrays

.filter()

.filter() goes through every item and keeps only the ones that pass a condition — returning a smaller array.

📖 How it works

If the condition returns true → item is kept. If false → item is removed.

Original array is unchanged. New array may be shorter.

const ages = [12, 20, 16, 25, 14];

// Keep only adults (18+)
const adults = ages.filter((age) => age >= 18);
console.log(adults); // [20, 25]

// Keep only short words
const words  = ["hi", "hello", "ok", "goodbye"];
const short  = words.filter((w) => w.length <= 3);
// ["hi", "ok"]

Lesson 8 · Arrays

.find()

.find() returns the first item that matches a condition. If nothing matches, it returns undefined.

📖 .find() vs .filter()

.filter() → returns an array of all matches.

.find() → returns the first match only (one item, not an array).

const users = [
  { id: 1, name: "Alice" },
  { id: 2, name: "Bob" },
  { id: 3, name: "Charlie" },
];

const found = users.find((u) => u.id === 2);
console.log(found); // { id: 2, name: "Bob" }

const none = users.find((u) => u.id === 99);
console.log(none);  // undefined

Lesson 9 · Arrays

.some() & .every()

These two methods check conditions across an array and return true or false.

📖 How it works

.some() → at least one item passes? Returns true.

.every() → all items pass? Returns true.

const ages = [17, 20, 15, 22];

ages.some((a) => a >= 18);   // true  (20 and 22 pass)
ages.every((a) => a >= 18);  // false (17 and 15 fail)

const scores = [80, 90, 85];
scores.every((s) => s >= 70); // true  (all pass!)

Lesson 10 · Objects

Object Basics

An object groups related data together using key: value pairs — like a profile card for one thing.

📖 How it works

Access values with object.key (dot notation) or object["key"].

You can add, change, or delete properties at any time.

const user = {
  name:  "John",
  age:   25,
  email: "john@gmail.com",
};

user.name;         // "John"
user["age"];      // 25

user.age = 26;    // update a value
user.city = "NY"; // add a new key
delete user.email; // remove a key

Lesson 11 · Objects

Array of Objects

Most real-world data is an array of objects — like a list of users from a database. Combine everything you learned!

📖 How it works

Use .map() to transform, .filter() to narrow down, and .find() to locate one item — all on arrays of objects.

const users = [
  { name: "Alice", age: 22, active: true },
  { name: "Bob",   age: 17, active: false },
  { name: "Carol", age: 30, active: true },
];

// Get names of active adults
const result = users
  .filter((u) => u.active && u.age >= 18)
  .map((u) => u.name);

console.log(result); // ["Alice", "Carol"]
